# smollerlm-for-sqlite

For those weirdos who see [llama2.c](https://github.com/karpathy/llama2.c) and think "Hmmm... what if it ran entirely in SQLite?".

![](https://raw.githubusercontent.com/Daviljoe193/smollerlm-for-sqlite/refs/heads/main/just-an-llm-where-it-doesnt-belong.png?token=GHSAT0AAAAAADUBPMTU37QBCJ75BP4R3FXA2NSEUEA)

___

### Useage

First, you need the dependencies (In a VENV, silly c:) to create the database.

`pip install torch transformers`

And then, you need to run

`python db_build.py`

for the default `Explain what an SQL database is.` prompt with 400 token cap, or...

`python db_build.py "What is the meaning of life?"`

for a custom prompt with 400 token cap, or...

`python db_build.py "What is the meaning of life?" 500`

...to do the same as previous, but with a 500 token cap. You can also add `-c` to the end to copy the `.sql` stuff to the clipboard instead.

Now you can finally do the dirty and run it. Just load that `.sql` file into whatever, or paste the command generated by the python script into your tool of choice. In my case, I specifically made this with DB Browser for SQLite in mind, but it also works just as well in `sqlite3` in the terminal.

That's... it, really. The default inference params are a temperature of 0.8, a top_p of 0.9, a top_k of 40, a min_p of 0.1, and a repetition penalty of 1.2 (At 10 million parameters, SmollerLM 10M really needs this).
